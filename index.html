Die Reise eines Datenpakets

Das Internet ist kein direkter Draht von dir zum Ziel. Deine Daten reisen als Paket von Station zu Station – über Router und Knotenpunkte –, bis sie ankommen. Schick ein Paket los und schau zu. Klicke eine Leitung an, um sie zu kappen (wie ein Kabelbruch) – das Paket sucht sich dann selbst einen Umweg.

Das Netzwerk

Stationen sind Knoten, Leitungen verbinden sie. Klicke eine Leitung, um sie zu kappen – oder eine Station, um sie abzuschalten.

    Wie findet das Paket seinen Weg? In drei Punkten

    1. Kein direkter Draht. Deine Daten werden in ein Paket verpackt. Es gibt keine direkte Leitung zum Ziel – nur viele Stationen, die untereinander verbunden sind.
    2. Hop für Hop weiterreichen. Jede Station schaut: „Über welchen Nachbarn komme ich am schnellsten zum Ziel?" – und reicht das Paket dorthin weiter. Dieses Weiterreichen nennt man einen Hop.
    3. Umweg bei Ausfall oder Stau. Fällt eine Leitung aus oder ist eine Station überlastet (zu hohes Datenaufkommen), wird sie umgangen: Das Netz sucht automatisch einen anderen Weg. Genau diese Robustheit macht das Internet so stabil.
    Fachbegriffe: Das Berechnen des Weges heißt Routing. Weil sich der Weg an die aktuelle Lage anpasst (Ausfälle, Überlastung), spricht man von dynamischem Routing. Eine überlastete Station zu meiden ist Stauvermeidung – wie eine Navi-App, die einen Stau umfährt. Der kürzeste Weg (wenigste Hops) wird hier mit der Breitensuche gefunden. Dass es mehrere Wege gibt, ist Absicht: So übersteht das Internet sogar Ausfälle einzelner Stationen.
